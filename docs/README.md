# 💭 기능 요구 사항 분석

**📌 주어진 횟수 동안 n대의 자동차는 전진 또는 멈출 수 있다.**
- [ ] 반복 횟수(게임 시도 횟수)를 입력 받는다.
- [ ] 자동차 대수를 입력 받는다.
- [ ] 자동차가 전진한다.
- [ ] 자동차가 정지한다.

**📌 각 자동차에 이름을 부여할 수 있다. 전진하는 자동차를 출력할 때 자동차 이름을 같이 출력한다.**
- [ ] 자동차에 이름을 부여한다.
- [ ] 자동차의 이름을 출력한다.
- [ ] 전진하는 자동차를 식별하고 해당 자동차의 이름을 출력한다.

**📌 자동차 이름은 쉼표(,)를 기준으로 구분하며 이름은 5자 이하만 가능하다.**
- [ ] 쉼표를 구분자로 사용하여 자동차의 이름을 구분한다.
  - [ ] 이름이 중복되면 예외가 발생한다.
- [ ] 자동차 이름은 5자 이하로 제한한다. 글자 수가 넘어가면 예외가 발생한다.
  - [ ] 공백이 입력되었다면 예외가 발생한다.

**📌 사용자는 몇 번의 이동을 할 것인지를 입력할 수 있어야 한다.**
- [ ] 사용자는 이동 횟수(반복 횟수)를 입력할 수 있다.

**📌 전진하는 조건은 0에서 9 사이에서 무작위 값을 구한 후 무작위 값이 4 이상일 경우이다.**
- [ ] 0에서 9 사이의 무작위 값을 구했을 때, 값이 4 이상일 때만 전진한다.
  - 4 이상의 수가 나왔을 때 1전진 되었는지 검증한다.

**📌 자동차 경주 게임을 완료한 후 누가 우승했는지를 알려준다. 우승자는 한 명 이상일 수 있다.**
- [ ] 기록된 점수를 바탕으로 우승자를 판별한다(동점자 처리).

**📌 우승자가 여러 명일 경우 쉼표(,)를 이용하여 구분한다.**
- [ ] 구분자를 쉼표로 우승자를 구분하여 출력한다.
  - 정규 표현식을 사용해보자.

**📌 사용자가 잘못된 값을 입력할 경우 `IllegalArgumentException`을 발생시킨 후 애플리케이션은 종료되어야 한다.**
- [ ] 입력 예외 케이스에 맞게 적절한 Exception이 발생한다.
- [ ] Exception 발생 후 애플리케이션은 종료한다.
